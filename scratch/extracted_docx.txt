§1 — Overview &amp; Architecture
Forge AI Builder is a Next.js 14 application where the backend lives entirely within Next.js Route Handlers (App Router). There is no separate Express or NestJS server. All backend logic — Salesforce OAuth, AI chat, deployment, Jira, Git, and billing — runs as serverless functions co-located with the frontend on Vercel.
1.1  Backend Architecture Principles
Serverless-first — all API routes are Next.js Route Handlers deployed as Vercel Edge/Serverless Functions
Org-isolated — every Supabase query is scoped to the authenticated user_id via Row-Level Security (RLS)
Async deployment — Salesforce Metadata API calls are long-running (45–90s); handled with Supabase Realtime streaming
Atomic action counter — Redis INCR prevents race conditions on subscription plan limits
AI prompt caching — org metadata injected into every Claude call; prompt caching reduces API cost 35–45%

§5 — Deployment Engine
5.1  POST /api/deploy — Approve &amp; Deploy
This is the most critical Route Handler. It is only reachable after the human approval step in the UI. The middleware verifies the approval record exists with a human timestamp before any JSforce call is made.
Step
Service
Detail
1 — Verify approval
Supabase deployment_approvals
Check approved_by IS NOT NULL and approved_at is within 30 minutes. Block if missing.
2 — Create deployment record
Supabase deployments
INSERT with status=deploying, git_branch, jira_ticket_id. Returns deployment_id.
3 — For each plan step
JSforce + Supabase Realtime
Execute step → write result to deployment_steps table → Supabase Realtime broadcasts to frontend.
4 — Apex steps
Tooling API ContainerAsyncRequest
Compile Apex → check CompilationResult → run tests → check coverage &gt;= 75%. Auto-rollback if tests fail.
5 — Metadata steps
JSforce metadata.deploy()
Package XML → deploy() → poll DeployResult every 3 seconds → write status updates to Realtime.
6 — All steps done
lib/ai.ts selfEvaluate()
Claude verifies each acceptance criterion via follow-up Salesforce API queries. Returns 0-100 confidence score.
7 — Store evaluation
Supabase deployments
UPDATE deployments SET acceptance_criteria_results, confidence_score, status=success.
8 — Rollback metadata
JSforce metadata.retrieve()
Retrieve pre-deployment state of changed items → store as JSONB in deployments.rollback_metadata. Available 24h.
5.2  Supabase Realtime — Live Deployment Log
The frontend subscribes to a Supabase Realtime channel on the deployment_steps table, filtered by deployment_id. Each step completion is a DB INSERT that Realtime broadcasts in real time.
Frontend Subscription Pattern
const channel = supabase.channel(&apos;deployment:&apos; + deploymentId)
  .on(&apos;postgres_changes&apos;, { event: &apos;INSERT&apos;, schema: &apos;public&apos;, table: &apos;deployment_steps&apos;,
       filter: &apos;deployment_id=eq.&apos; + deploymentId }, (payload) =&gt; {
    updateDeploymentLog(payload.new);  // renders step card in UI
  }).subscribe();
5.3  DELETE /api/deploy/[id]/rollback — Rollback
Step
Action
